---
title: Delete Category
api: DELETE /categories/:id
---

Deletes a category from the system.

<Note>
  This endpoint requires authentication with an `ADMIN` role.
</Note>

<Warning>
  Deleting a category may fail if there are products associated with it due to foreign key constraints.
</Warning>

## Authentication

Requires a valid JWT token in the Authorization header:
```
Authorization: Bearer <token>
```

## Path Parameters

<ParamField path="id" type="number" required>
  Category ID (must be numeric)
</ParamField>

## Response

<ResponseField name="data" type="Category">
  The deleted category object
  
  <Expandable title="Category object">
    <ResponseField name="id" type="number" required>
      Unique identifier for the category
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Category name
    </ResponseField>
    
    <ResponseField name="slug" type="string" required>
      URL-friendly slug
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean" required>
      Whether the category is active
    </ResponseField>
    
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the category was created
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl --request DELETE \
  --url https://api.example.com/categories/1 \
  --header 'Authorization: Bearer <token>'
```
</RequestExample>

<ResponseExample>
