---
title: Get Category
api: GET /categories/:identifier
---

Retrieves a single category by its ID or slug.

## Path Parameters

<ParamField path="identifier" type="string" required>
  Category ID (numeric) or slug (string). The endpoint automatically detects whether the identifier is numeric or a slug.
</ParamField>

## Response

<ResponseField name="data" type="Category">
  Category object
  
  <Expandable title="Category object">
    <ResponseField name="id" type="number" required>
      Unique identifier for the category
    </ResponseField>
    
    <ResponseField name="name" type="string" required>
      Category name
    </ResponseField>
    
    <ResponseField name="slug" type="string" required>
      URL-friendly slug, automatically generated from name
    </ResponseField>
    
    <ResponseField name="isActive" type="boolean" required>
      Whether the category is active
    </ResponseField>
    
    <ResponseField name="createdAt" type="string" required>
      ISO 8601 timestamp of when the category was created
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash By ID
curl --request GET \
  --url https://api.example.com/categories/1
```

```bash By Slug
curl --request GET \
  --url https://api.example.com/categories/teas
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 1,
  "name": "Teas",
  "slug": "teas",
  "isActive": true,
  "createdAt": "2024-03-10T12:00:00.000Z"
}
```

```json 404
{
  "statusCode": 404,
  "message": "Category not found"
}
```
</ResponseExample>
